# トップページ（Index）リデザイン実装計画

## ユーザーレビューが必要な項目
- **スライドショー画像**: ImgurやUnsplashのプレースホルダー画像を使用します。後ほど1920x1080の実画像に差し替えてください。
- **メンバー詳細**: 3名分のプレースホルダーテキスト/画像を使用します。
- **Notifyロジック**: 言語不一致（例：ブラウザ=JP、ページ=EN -> JPを提案）のクライアントサイドチェックを実装します。

## 変更案

### [layouts/BaseLayout.astro]
- 変更なし。既存を再利用します。

### [pages/[lang]/index.astro]
- **構造の刷新**:
    1. **バナースライドショー**: 新規実装（インラインHTML/Scriptまたはコンポーネント）。
    2. **サイトタイトル**: 既存の `greeting/description` を使用。
    3. **Notify**: タイトルとチーム紹介の間に新セクションを追加。
    4. **チーム紹介**: `.content-section`, `.text-column`, `.image-column` クラスを再利用。
    5. **メンバー紹介**: 新規「SNS風」カードレイアウトを作成。
    6. **作品概要**: 2カラムレイアウトを再利用。
    7. **サイト紹介**: 2カラムレイアウトを再利用。
- **CSS**: `<style>`ブロックにスライドショー、Notify、メンバーカードのスタイルを追加します。

### [Components]
- 複雑になる場合は `components/Slideshow.astro` を作成しますが、単一利用のためまずはインラインでシンプルに実装します。

## 検証計画

### 自動テスト
- UIレイアウト用の既存テストはないため、`astro dev` で表示を確認します。

### 手動検証
- **スライドショー**: 自動スライド、ナビゲーション（前後プレビュー）の動作確認。
- **Notify**: ブラウザの言語設定を変更し、適切な言語切り替え提案が表示されるか確認。
- **2カラムレイアウト**: ブログ記事などの既存レイアウトと整合性が取れているか確認。
- **レスポンシブ**: モバイル表示でのセクションの積み重なり方などを確認。
