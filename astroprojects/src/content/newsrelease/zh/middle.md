# Shape Writer

“看清”信息的流动

![中間報告1](https://img.z4a.net/images/2026/02/14/middle_1.png)

# 主题：信息流

我们通常只关注数据的最终结果——接收或存储——而很少关注数据通过优先级排序或网络传输的过程。

当数据从电脑保存到硬盘时，数据必须以电信号的形式在电缆内进行物理传输。

Shape Writer旨在以可视化的方式呈现这种数据流。

![中間報告2](https://img.z4a.net/images/2026/02/14/middle_2.png)

# 操作流程

1. 根据输入信息挤出字母
2. 收集在收集盒中
3. 沿传送带流动
4. 存储

您可以观察输入信息的“流动”过程。

![中間報告3](https://img.z4a.net/images/2026/02/14/middle_3.png)

# 字盒

使用伺服电机，一次推出一个字符位置的字母。

每个字符类型都准备了一个。

![中間報告5](https://img.z4a.net/images/2026/02/14/middle_5.png)

# 附录

由于“发送”功能是主要的，因此信息存储部分没有添加恢复功能。

**安装吸引力**

- 参与者只能在字符出现在传送带上时感知到信息（按下 Enter 键时，UI 也会消失）
- 您可以知道有人在之前输入了字符（但不知道是什么）

字符会堆积在盒子里

可能像“容量”仪表一样有趣

**视觉解释**

数据传输期间的进度条灵感

字符是信息本身

字符作为概念“信息”的图标
